# NoteWizard 模块文档：安全与加密 (Security)

## 1. 加密架构 (Recovery Key System)
- **底层算法**: AES-256-GCM (Authenticated Encryption)。
- **全局加密状态**: 加密是一种“工作区级别”的状态。一旦启用加密，数据库中的所有笔记内容及其对应的媒体文件（images/objects）均会以密文存储。加密文件的格式为：`ENCRYPTED:v1:salt:iv:authTag:data`。
- **恢复密钥 (Recovery Key)**: 
  - 系统生成一组 12 段由 4 位字符组成的密钥（例如：`abcd-efgh-...`）。
  - 该密钥通过 `PBKDF2` 算法派生出实际的 AES 密钥。
  - **重要**: 恢复密钥是找回数据的唯一凭证。

## 2. 密钥保护与解锁
- **Electron SafeStorage**: 
  - 恢复密钥在本地通过 Electron 的 `safeStorage` API 进行二级加密。
  - `safeStorage` 依赖操作系统层级的凭据（如 Windows DPAPI 或 macOS Keychain），确保即使数据库文件被拷贝到其他电脑，也无法在未授权的情况下解密。
- **内存安全**:
  - 解密后的原始密钥仅在应用运行期间保留在主进程内存中。
  - 应用关闭或崩溃后，内存中的密钥立即销毁。

## 3. 导出安全性与全量导出
- **全量导出 (.nwp)**: 导出为项目专属格式时，必须手动输入“恢复密钥”进行校验，加密文件的密文会被解密为明文。
- **导出恢复**: 在其他设备导入或尝试导出为 Markdown 明文时，必须手动输入“恢复密钥”进行校验。

## 4. 本地化隔离
- **物理路径匿名**: 加密后的笔记内容存储于 `objects/` 目录，文件名采用哈希或随机 ID，不包含任何原始标题信息，防止元数据泄露。
