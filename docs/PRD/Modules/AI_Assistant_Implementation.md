# AI 辅助写作功能实现说明

## 功能概述

AI辅助写作功能为用户提供智能续写建议，类似于 GitHub Copilot 的体验。

## 核心特性

### 1. 灰色预览提示
- AI生成的补全内容以灰色、斜体形式显示在光标位置
- 不会干扰用户的正常输入
- 视觉上清晰区分建议内容和实际内容

### 2. 交互方式
- **Tab 键**：接受AI建议，将补全内容插入到文档中
  - 有AI建议时：接受建议
  - 无AI建议时：执行正常的缩进功能
- **Esc 键**：拒绝AI建议，清除预览
- **继续输入**：自动清除当前建议，触发新的补全请求

> 注意：Tab 键的行为是智能的 - 只有在显示AI建议时才会接受建议，否则执行正常的编辑器缩进功能

### 3. 智能触发
- 用户停止输入后延迟触发（默认2秒，可配置）
- 最小输入长度限制（默认10字符，可配置）
- 自动使用光标前500字符作为上下文
- **连续补全**：接受建议后自动触发下一次补全（300ms短延迟）

### 4. 可配置选项
在首选项 > 文思泉涌 中可配置：
- **启用/禁用**：开关AI写作助手
- **AI服务源**：选择使用的AI服务
- **输入延迟**：设置触发补全的等待时间（毫秒）
- **最小输入长度**：设置触发补全的最小字符数
- **系统提示词**：自定义AI的行为和风格

## 技术实现

### 架构组件

#### 1. 主进程 (Electron Main)
- **文件**: `electron/main/ipc/modules/ai-chat.js`, `electron/main/services/ai-config.service.js`, `electron/main/services/remote-ai.service.js`
- **功能**: 
  - 处理AI补全请求
  - 在主进程内读取并解析 AI Assistant 配置
  - 统一解析 AI Source、模型和系统提示词
  - 请求超时控制（30秒）
  - 输入验证和错误处理
  - 调用 OpenAI 兼容 API
  - 返回补全结果

#### 2. 编辑器插件
- **文件**: `src/renderer/core/editor/aiCompletionPlugin.ts`
- **功能**:
  - 定义补全建议的状态管理（StateField）
  - 实现灰色预览的Widget
  - 处理Ctrl+Enter/Esc快捷键
  - 管理建议的显示和清除
  - 自动清除建议（用户输入时）

#### 3. Composable
- **文件**: `src/renderer/features/editor/composables/useAiAssistant.ts`
- **功能**:
  - 每个编辑器实例独立管理状态
  - 处理输入延迟逻辑（防抖）
  - 请求取消机制（AbortController）
  - 超时控制（30秒）
  - 调用IPC请求AI补全
  - 显示/清除补全建议
  - 资源清理

#### 4. 编辑器集成
- **文件**: `src/renderer/features/editor/components/EditorPane.vue`
- **功能**:
  - 监听用户输入事件
  - 触发AI补全请求
  - 响应配置变化
  - 生命周期管理

### 数据流

```
用户输入 
  → EditorPane.onChange (isAiCompletion=false)
  → useAiAssistant.handleTyping (延迟2秒)
  → aiService.generateCompletion
  → electronApi.aiChat.generateCompletion
  → IPC: AI_CHAT_GENERATE_COMPLETION
  → ai-chat.js / ai-config.service.js (主进程)
  → remote-ai.service.js
  → 返回补全结果
  → showAiSuggestion (显示灰色预览)
  → 用户按Tab接受
  → acceptSuggestion (标记为AI补全)
  → EditorPane.onChange (isAiCompletion=true)
  → useAiAssistant.requestCompletion (短延迟300ms)
  → 循环：显示下一个建议
```

## 配置示例

### 默认系统提示词
```
你是一位AI写作助手，请根据用户提供的上下文，续写或完善文本。只需要返回续写的部分，不要重复用户的文本。每次最多返回一句话。
```

### 自定义系统提示词示例

**技术文档助手**:
```
你是一位专业的技术文档写作助手。请根据上下文续写技术文档，保持专业、准确、简洁的风格。使用清晰的技术术语，每次返回一个完整的句子或段落。
```

**创意写作助手**:
```
你是一位富有创意的小说作家。请根据故事情节续写，保持叙事的连贯性和吸引力。注重细节描写和情感表达，每次返回一到两句话。
```

**学术论文助手**:
```
你是一位学术写作助手。请根据论文内容续写，使用正式的学术语言，保持逻辑严谨和论证充分。每次返回一个完整的学术句子。
```

## 样式定制

AI补全建议的样式在 `src/renderer/styles/editor.css` 中定义：

```css
.cm-ai-suggestion {
  color: #888;
  opacity: 0.6;
  font-style: italic;
  pointer-events: none;
  user-select: none;
}
```

可以根据需要调整颜色、透明度和字体样式。

## 使用建议

1. **调整延迟时间**：如果觉得触发太快或太慢，可以在首选项中调整"输入延迟"
2. **自定义提示词**：根据写作类型（技术文档、创意写作、学术论文等）定制系统提示词
3. **选择合适的模型**：不同的AI模型有不同的特点，可以尝试不同的模型找到最适合的
4. **最小输入长度**：设置合理的最小输入长度，避免在输入很少内容时就触发补全
5. **连续补全**：接受建议后会自动触发下一次补全，可以连续按Tab快速生成内容

## 注意事项

1. 需要先在"AI配置"中添加并测试AI服务源
2. 确保选择的AI服务源支持OpenAI兼容的API格式
3. AI补全会消耗API调用次数，请注意使用成本
4. 补全质量取决于所选的AI模型和系统提示词的设置


## 最佳实践与优化

### 1. 性能优化
- ✅ **配置缓存**：主进程缓存配置5秒，避免频繁读取文件
- ✅ **请求取消**：用户快速输入时自动取消之前的请求
- ✅ **防抖处理**：延迟触发补全，避免过多请求
- ✅ **独立状态**：每个编辑器实例独立管理状态，避免冲突

### 2. 用户体验
- ✅ **智能快捷键**：Tab键智能处理 - 有建议时接受，无建议时缩进
- ✅ **连续补全**：接受建议后自动触发下一次补全，实现流畅的写作体验
- ✅ **自动清除**：用户继续输入时自动清除旧建议
- ✅ **视觉反馈**：灰色斜体显示，清晰区分建议和实际内容
- ✅ **国际化支持**：默认提示词根据语言自动切换

### 3. 错误处理
- ✅ **输入验证**：检查上下文长度和有效性
- ✅ **超时控制**：30秒超时，避免长时间等待
- ✅ **错误提示**：清晰的错误消息
- ✅ **优雅降级**：API失败时不影响编辑器正常使用

### 4. 资源管理
- ✅ **定时器清理**：组件卸载时清理所有定时器
- ✅ **请求取消**：使用AbortController取消未完成的请求
- ✅ **内存管理**：使用shallowRef避免不必要的响应式开销

### 5. 安全性
- ✅ **输入限制**：上下文最大10000字符
- ✅ **API密钥保护**：密钥仅在主进程中使用
- ✅ **请求验证**：验证所有输入参数

## 已知限制

1. **单行补全**：当前实现每次只返回一句话，适合逐句写作
2. **上下文长度**：固定使用光标前500字符，未来可配置
3. **无流式输出**：使用非流式API，未来可支持流式显示
4. **语言检测**：系统提示词根据应用语言，不根据文档内容

## 未来改进方向

1. **流式输出**：支持流式API，实时显示生成内容
2. **多行补全**：支持生成多行内容
3. **智能上下文**：根据文档类型调整上下文长度
4. **语言检测**：自动检测文档语言，使用对应的提示词
5. **补全历史**：记录补全历史，支持撤销/重做
6. **快捷键自定义**：允许用户自定义接受/拒绝的快捷键
7. **补全质量评分**：收集用户反馈，优化补全质量
